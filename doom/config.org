* IJKL Navigation Setup

This section remaps the core Vim movement keys from =hjkl= to the =ijkl=
"inverted-T" layout and carefully relocates all displaced keys.

** Keybinding Reference

| Key | Normal          | Visual          | Insert  | Operator          | Motion/Minibuf |
|-----+-----------------+-----------------+---------+-------------------+----------------|
| i   | Up              | Extend up       | type i  | (text obj: inner) | type i         |
| k   | Down            | Extend down     | type k  | (unchanged)       | type k         |
| j   | Left            | Extend left     | type j  | (unchanged)       | type j         |
| l   | Right           | Extend right    | type l  | (unchanged)       | type l         |
| ;   | Enter insert    | —               | type ;  | —                 | type ;         |
| :   | Insert at BOL   | —               | type :  | —                 | type :         |
| K   | Join lines      | Join lines      | —       | —                 | —              |
| u   | Undo            | Undo            | —       | —                 | —              |
| U   | Redo            | Redo            | —       | —                 | —              |
| M-; | Ex command      | Ex command      | —       | —                 | —              |

** Core Navigation (Normal, Visual)

Movement keys are bound in =normal= and =visual= states only.
- *Operator-pending* is excluded so text objects (=ciw=, =di(=, =yi"=) work.
- *Motion state* is excluded so minibuffer/M-x/Vertico typing works normally.

#+begin_src elisp
(after! evil
  (defun +custom/apply-ijkl-core-bindings ()

    ;; Core movement in normal + visual states.
    (define-key evil-normal-state-map (kbd "i") #'evil-previous-line)
    (define-key evil-normal-state-map (kbd "k") #'evil-next-line)
    (define-key evil-normal-state-map (kbd "j") #'evil-backward-char)
    (define-key evil-normal-state-map (kbd "l") #'evil-forward-char)
    (define-key evil-visual-state-map (kbd "i") #'evil-previous-line)
    (define-key evil-visual-state-map (kbd "k") #'evil-next-line)
    (define-key evil-visual-state-map (kbd "j") #'evil-backward-char)
    (define-key evil-visual-state-map (kbd "l") #'evil-forward-char)

    ;; Displaced keys.
    (define-key evil-normal-state-map (kbd ";") #'evil-insert)
    (define-key evil-normal-state-map (kbd ":") #'evil-insert-line)
    (define-key evil-normal-state-map (kbd "K") #'evil-join)
    (define-key evil-visual-state-map (kbd "K") #'evil-join)
    (define-key evil-normal-state-map (kbd "U") #'evil-redo)
    (define-key evil-visual-state-map (kbd "U") #'evil-redo)
    (define-key evil-normal-state-map (kbd "M-;") #'evil-ex)
    (define-key evil-visual-state-map (kbd "M-;") #'evil-ex))

  ;; Apply once now, then re-apply after Doom modules load to win precedence.
  (+custom/apply-ijkl-core-bindings)
  (add-hook 'doom-after-modules-config-hook #'+custom/apply-ijkl-core-bindings))
#+end_src

** Window Management (SPC w / C-w)

Doom binds =SPC w= to =evil-window-map= by default, which contains all the
standard window operations (split, close, resize, rotate, etc.). We add our
ijkl navigation directly into that existing map so nothing is lost.

#+begin_src elisp
(defun +custom/window-top-right ()
  "Select the top-right window in the current frame."
  (interactive)
  (let ((best (selected-window))
        (best-right -1))
    (walk-windows
     (lambda (w)
       (let ((edges (window-edges w)))
         (when (and (= (nth 1 edges) 0)            ;; top row
                    (> (nth 2 edges) best-right))   ;; rightmost
           (setq best-right (nth 2 edges)
                 best w))))
     nil (selected-frame))
    (select-window best)))

(map! :map evil-window-map

      "k" #'evil-window-down
      "j" #'evil-window-left

      "K" #'+evil/window-move-down
      "J" #'+evil/window-move-left
      "L" #'+evil/window-move-right

* Eglot: Keep JSON files in json-mode

Doom's tree-sitter module remaps =json-mode= to =json-ts-mode=.  Override via
=major-mode-remap-alist= so =.json= files stay in plain =json-mode=.

Even with the mode fixed, Doom's =:lang javascript= module configures eglot to
attach =typescript-language-server= to =json-mode= buffers, producing bogus
=; expected= diagnostics.  Explicitly reassign =json-mode= in
=eglot-server-programs= to =vscode-json-language-server= so the correct server
is used.  Requires =vscode-json-language-server= (=npm i -g vscode-langservers-extracted=).

#+begin_src elisp
(add-to-list 'major-mode-remap-alist '(json-ts-mode . json-mode))

    '("vscode-json-language-server" "--stdio")))
#+end_src

The buffer-local =apheleia-formatter= (aliased to =+format-with= in Doom) is

*** Agent Shell (AI coding agent)

Uses [[https://github.com/xenodium/agent-shell][agent-shell]] to interact with AI coding agents via ACP (Agent Client
Protocol) natively in Emacs, instead of launching them in a vterm split.

Detects the machine by checking which Doom config directory exists -- the same
logic used in the push/pull scripts. Personal laptop uses =~/.doom.d= (Claude
Agent), work laptop uses =~/.config/doom= (OpenCode with Cloudflare SSO).

(use-package! agent-shell

             agent-shell-new-shell
             agent-shell-opencode-start-agent
             agent-shell-anthropic-start-claude-code)

  ;; Personal (~/.doom.d): Claude Agent with login-based auth
  ;; Work (~/.config/doom): OpenCode with no API key (uses `opencode auth login` / Cloudflare SSO)
  (if (file-directory-p (expand-file-name "~/.doom.d"))
      (progn
        (setq agent-shell-preferred-agent-config 'claude-code)
        (after! agent-shell-anthropic
          (setq agent-shell-anthropic-authentication
                (agent-shell-anthropic-make-authentication :login t))))
    (progn
      (setq agent-shell-preferred-agent-config 'opencode)
      (after! agent-shell-opencode
        (setq agent-shell-opencode-authentication
              (agent-shell-opencode-make-authentication :none t)))))

  :config
  (require 'acp)

  ;; ---------------------------------------------------------------------------

  ;; ---------------------------------------------------------------------------
